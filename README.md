# English Flux Academy Landing

Продающий лендинг онлайн-курсов английского языка, разработанный как портфолио-проект под реальные бизнес-задачи.

## Что решает этот проект:

### Задача
- Увеличить количество заявок на обучение через сайт.
- Сократить путь пользователя от первого экрана до отправки формы.
- Повысить конверсию за счет понятного оффера, социального доказательства и сильного CTA.

### Решение
- Разработан быстрый адаптивный лендинг с акцентом на лидогенерацию.
- Реализована многошаговая структура доверия: Hero -> Программы -> Отзывы -> Тариф -> FAQ -> Форма.
- Добавлены интерактивные сценарии, повышающие вовлечение:
  - плавные reveal-анимации блоков при скролле,
  - «плавающие» 3D-карточки в hero,
  - адаптивное бургер-меню,
  - sticky mobile CTA.
- Форма захвата лидов с валидацией email и телефона + состояния отправки (loading/success/error).
- Светлая и темная темы для современного UX и лучшей визуальной адаптации под устройство пользователя.

### Результат (по текущей реализации)
- Производительность фронтенда:
  - production build за ~1.31s,
  - итоговый JS-бандл: ~18.96 kB (6.71 kB gzip),
  - итоговый CSS-бандл: ~18.91 kB (4.15 kB gzip).
- Подготовлена MPA-структура для SEO и рекламного трафика:
  - `index.html`, `about.html`, `404.html`.
- Реализована готовая база для интеграции с CRM/вебхуками и масштабирования лидогенерации.

## Почему такой подход выгоден бизнесу

- **Быстрый запуск рекламного трафика:** легкий фронт, понятная структура, готовые CTA.
- **Высокая гибкость контента:** тексты, тарифы, блоки легко менять без пересборки архитектуры.
- **Чистая компонентная база:** проще поддержка, дешевле доработки, ниже риски регрессий.
- **Подготовка к росту:** легко добавить аналитику, A/B тесты, CRM-интеграции, мультилендинг.

## Техническая часть

### Стек
- HTML5 (MPA)
- Vanilla JavaScript (ES Modules)
- SCSS
- Vite 4

### Подходы разработки
- Компонентная структура (`components / layouts / sections / modules / pages`).
- BEM-нейминг для предсказуемой стилизации.
- Mobile-first адаптация + fluid-подход:
  - `fluid()`
  - `fluid-to-laptop()`
- Доступность и UX:
  - `aria-label`, корректные `button/link` сценарии,
  - `prefers-reduced-motion` для анимаций,
  - валидация форм с пользовательской обратной связью.

### Архитектура
- Точка входа: `src/main.js`
- Сборка стилей: `src/appStyles.js`
- Роутинг по `pathname` для MPA-страниц
- Поведенческие модули:
  - переключение темы,
  - accordion FAQ,
  - reveal on scroll,
  - управление мобильным меню,
  - логика формы лида,
  - скрытие mobile CTA внизу страницы.
